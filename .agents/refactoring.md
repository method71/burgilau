# План рефакторинга Burgilau

> Временная инструкция. После выполнения всех этапов этого плана удалите этот
> файл и соответствующую запись из корневого `AGENTS.md`. Перед удалением
> перенесите сохраняющие актуальность правила в постоянные тематические
> инструкции.

Работа выполняется короткими завершёнными этапами. После каждого этапа проверяются сборка, функциональность и внешний вид, затем изменения можно зафиксировать отдельным коммитом.

## Рабочий процесс

1. Выбрать один этап.
2. Проверить текущее состояние Git и сохранить посторонние изменения.
3. Выполнить рефакторинг в рамках выбранного этапа.
4. Запустить доступные автоматические проверки.
5. Проверить страницу визуально в браузере.
6. Исправить найденные отклонения.
7. Зафиксировать завершённый этап отдельным коммитом.

## Согласованные правила именования

- Для шапки используется блок `header` без префикса `site-`.
- Для калькулятора используется сокращённый блок `calc`.

Примеры:

```text
site-header           → header
site-logo             → header__logo
site-brand            → header__brand
site-location         → header__location
site-location-icon    → header__location-icon

calculator            → calc
field                 → calc__field
field-title           → calc__field-title
options               → calc__options
cta                   → calc__submit
cta-icon              → calc__submit-icon

option                → option-card
option-icon           → option-card__icon
option-check          → option-card__check

estimate-price        → estimate__price
estimate-note         → estimate__note

master                → company-card
master-photo          → company-card__logo
master-name           → company-card__name
master-role           → company-card__description
master-arrow          → company-card__arrow
```

## Этап 1 — инфраструктура

- Подключить Vite и Sass.
- Добавить npm-скрипты для разработки и production-сборки.
- Организовать каталог `src`.
- Перенести исходники и ресурсы без изменения внешнего вида.
- Проверить production-сборку.

Команда для старта этапа:

> Начинай этап 1: подключи Vite и SCSS, организуй структуру `src`, сохрани текущий внешний вид и функциональность, проверь production-сборку. После выполнения остановись и отчитайся.

## Этап 2 — переход на БЭМ

- Провести аудит существующих классов.
- Подготовить итоговую таблицу переименований.
- Использовать основные блоки `header`, `hero`, `calc`, `option-card`, `estimate` и `company-card`.
- Синхронно изменить HTML и SCSS.
- Перевести JavaScript-селекторы на `data-*`.
- Не менять внешний вид и поведение страницы.

Команда для старта этапа:

> Начинай этап 2: переведи классы на БЭМ. Используй согласованные блоки и `data-*` для JavaScript. Сохрани текущий внешний вид и функциональность.

## Этап 3 — рефакторинг SCSS

- Разделить стили на модули.
- Удалить повторяющиеся и конфликтующие правила.
- Вынести цвета, размеры и интервалы в дизайн-токены.
- Создать единые брейкпоинты и необходимые миксины.
- Ограничить вложенность селекторов двумя-тремя уровнями.
- Не менять утверждённый дизайн.

Целевая структура:

```text
src/scss/
├── abstracts/
│   ├── _variables.scss
│   ├── _mixins.scss
│   └── _breakpoints.scss
├── base/
│   ├── _reset.scss
│   └── _typography.scss
├── layout/
│   ├── _page.scss
│   └── _header.scss
├── components/
│   ├── _hero.scss
│   ├── _calc.scss
│   ├── _option-card.scss
│   ├── _button.scss
│   └── _company-card.scss
└── main.scss
```

Команда для старта этапа:

> Начинай этап 3: раздели SCSS на модули, удали дублирование и конфликты, вынеси токены и брейкпоинты. Не меняй дизайн.

## Этап 4 — рефакторинг JavaScript

- Перевести JavaScript на ES-модули.
- Отделить конфигурацию цен и контактов от интерфейса.
- Создать чистую функцию расчёта стоимости.
- Отдельно реализовать форматирование цены.
- Отдельно реализовать генерацию WhatsApp-ссылки.
- Использовать `URLSearchParams` для формирования URL.
- Отделить работу с DOM от бизнес-логики.

Целевая структура:

```text
src/scripts/
├── config.js
├── calculator.js
├── whatsapp.js
├── ui.js
└── main.js
```

Команда для старта этапа:

> Начинай этап 4: раздели JavaScript на конфигурацию, расчёты, UI и WhatsApp. Переведи код на ES-модули и сохрани текущее поведение.

## Этап 5 — автоматические проверки

- Подключить ESLint для JavaScript.
- Подключить Stylelint с поддержкой SCSS.
- Подключить Prettier.
- Добавить unit-тесты бизнес-логики.
- Добавить браузерный тест основного пользовательского сценария.
- Настроить единую команду полной проверки проекта.

Предполагаемые команды:

```bash
npm run dev
npm run build
npm run lint
npm test
npm run check
```

Команда для старта этапа:

> Начинай этап 5: добавь ESLint, Stylelint для SCSS, Prettier и тесты бизнес-логики. Настрой общую команду проверки.

## Этап 6 — финальный аудит

- Проверить production-сборку.
- Проверить калькулятор для всех комбинаций параметров.
- Проверить WhatsApp-ссылку и кодирование кириллицы.
- Проверить мобильную и десктопную версии.
- Проверить управление с клавиатуры и доступность.
- Проверить производительность и отсутствие ошибок в консоли.
- Исправить найденные проблемы без изменения дизайна.

Команда для старта этапа:

> Начинай этап 6: проведи финальный аудит проекта — сборка, адаптивность, доступность, калькулятор и WhatsApp-ссылка. Исправь найденные проблемы без изменения дизайна.

## Критерий завершения рефакторинга

- Внешний вид и основная логика калькулятора сохранены.
- Проект собирается одной командой.
- SCSS организован по компонентам и не содержит конфликтующих дублей.
- HTML использует согласованный БЭМ-нейминг.
- JavaScript не зависит от CSS-классов.
- Цены и контакты находятся в конфигурации.
- Расчёты покрыты тестами.
- Линтеры и production-сборка проходят без ошибок.
